# DQN（深度 Q 网络）—— 用大白话讲清楚

> ⚠️ **阅读说明**：本文目标是让零 AI 背景的人听懂。为了好理解，有些地方做了简化，不完全严谨。

## 一句话说清楚 DQN 在干什么

**DQN 做的事：让 AI 自己学会打游戏。**

具体说：2013 年，DeepMind 做了一个神经网络，不给它任何游戏规则，只让它看屏幕像素和得分，它自己就学会了玩 Atari 游戏——而且在 3 款游戏上打得比人类还好。

这在当时是轰动性的成果。

---

## 类比：学打乒乓球

想象你从来没打过乒乓球：

**传统方法（不存在的完美世界）：**
有个教练手把手教你——"球往左来你就往左打，球往右来你就往右打，旋转球要这样接……" 这就是人工写规则。

**强化学习的基本思路：**
没有教练。你直接站到球桌前开始打。你只知道两个东西：
1. 当前状态——球飞到哪里了
2. 得分——赢了还是输了

你乱打一通，偶尔得分，慢慢摸索出规律："哦，球往左飞的时候我往左挥拍，得分的概率更高。" 这就是**从经验中学习**。

**DQN 的核心贡献：**
把上面这个"摸索规律"的过程，用一个神经网络自动化了。你不用手动总结规律，神经网络自己会从海量的"打球经验"中提炼出"在什么情况下做什么动作最好"。

## 几个关键概念的大白话解释

### Q 值 —— 就是"这一个动作能得多少分"的预期

Q 值的 Q 来自 "Quality"，但它本质上就是一个数字：**在目前的局面下，如果我做某个动作，平均下来最后能得多少分？**

比如玩打砖块（Breakout）：
- "球在左边 → 向左移" 这个动作的 Q 值 = 高分（好动作）
- "球在左边 → 向右移" 这个动作的 Q 值 = 低分（坏动作）

DQN 的神经网络就是用来**预测这些 Q 值**的。给它看当前画面，它输出每个动作的预期得分，选得分最高的那个执行。

### 神经网络 —— 就是一个巨大的"猜谜机器"

你可以把神经网络想象成一个**超级复杂的函数**。

你给它输入（比如游戏画面），它给你输出（每个动作的分数）。它内部由几千万个"旋钮"组成。训练的过程就是不断拧这些旋钮，让它的输出越来越准。

**一个比喻**：
- 传统程序员写代码：`if (球在左边) then 向左移` —— 人手工规定规则
- 神经网络：人不管规则，只给海量的"画面 → 正确动作"例子，网络自己总结规律

### Experience Replay（经验回放）—— 错题本

**问题**：AI 打游戏的经历是连续的——第 1 帧、第 2 帧、第 3 帧……高度相似。如果只拿最近的经验学习，它会"钻牛角尖"（过拟合）。

**解决方案**：准备一个大本子（回放缓冲区，能记 100 万条经验），把所有经历都记下来。每次学习的时候，**从这个大本子里随机翻几条出来学**，而不是只学最近的事。

就像你复习考试，不是只复习刚学的最后一章，而是从整本书里随机抽题做。

### Target Network（目标网络）—— 复读机

**问题**：DQN 的学习方式是"猜一个数，然后和目标的数比，不对就调整"。

但问题在于——**目标的数也在变**。因为用来算目标数的也是同一个神经网络。这就好比你在跑步，目标是前面的一个点，但你往前跑一步，那个点也往前挪一步——你永远追不上。

**解决方案**：做两个一样的网络：
- **主网络**：每天都在更新（你在跑）
- **目标网络**：固定住不动，每过 10000 步才从主网络复制一次（标记物是固定的）

这样标靶不再移动了，训练就稳定了。

## DQN 打游戏的全流程（用人话讲）

```
第 1 步：AI 看屏幕（4 帧画面堆叠在一起，能看出物体的运动方向）
第 2 步：AI 猜每个动作的得分（神经网络输出 Q 值）
第 3 步：大部分时候选得分最高的动作（贪心），偶尔随机乱试（探索）
第 4 步：执行动作，看到得分变化
第 5 步：把这次经历（画面→动作→得分→新画面）记到"错题本"里
第 6 步：从错题本里随机翻几条旧账来学习、调整
第 7 步：重复第 1-6 步，打几千万局

结果：AI 从完全不会，慢慢变成高手
```

## 为什么这个成果如此重要？

因为在这之前，做 AI 玩游戏有两种方法：
1. **手工写规则**（太累了，而且换一个游戏就得重写）
2. **传统机器学习**（需要人先提取"特征"——比如球的位置、速度等，再训练模型）

DQN 做到了**端到端学习**：只看原始像素，自己学会一切。输入就是画面，输出就是动作，中间不需要人插手"特征提取"和"策略设计"。

> ⚠️ **这里有个常见的误解需要澄清**——"端到端"不代表完全没有人参与。人仍然需要提供以下几个东西：

### 人需要提供的（不多，但关键）

| 需要人设计的 | 具体内容 | 类比 |
|------------|---------|------|
| **奖励系统（Reward Function）** | 什么算得分、什么算输 | 设计电子游戏的计分规则 |
| **动作空间（Action Space）** | AI 能做什么操作（左移、右移、发射等） | 设计游戏手柄有几个按钮 |
| **输入预处理** | 画面转灰度、缩到 84×84、裁掉无关区域、把最近 4 帧叠在一起 | 给 AI 配一副"眼镜"——让它看清该看的东西，挡住不该看的 |
| **神经网络结构** | 用几层卷积、每层多少个神经元 | 决定 AI 用多大一个"大脑"来思考 |
| **训练参数** | 学习率、探索策略（什么时候随机试探）、多久更新一次目标网络等 | 设定 AI 的学习习惯和节奏 |
| **游戏环境** | 用什么游戏、每局多久、是否跳过片头 | 选定 AI 的训练场 |

### 人不需要提供的（这才是突破所在）

不需要人做的恰恰是以前最费力的部分：

| 不需要人做的 | 以前怎么做 | DQN 怎么做 |
|------------|----------|-----------|
| **特征提取** | 人工写代码识别"球的位置、速度方向、挡板位置" | 神经网络自己从像素中学 |
| **策略规则** | `if 球.x < 挡板.x then 左移` 这种硬编码规则 | 神经网络自己从经验中总结 |
| **游戏知识** | 人先理解游戏机制，再翻译成代码 | 神经网络零基础开始，自己探索 |

### 关键区别

> **人设计的是 Harness（载具）—— 环境、控制器、计分板。**
> **模型学会的是 Agency（智能）—— 怎么判断、怎么决策、怎么玩。**
>
> 这正好呼应了 README 的核心框架。Harness ＝ 我们搭建的，Agency ＝ 模型学出来的。

所以你说的完全对：**没有奖励函数和动作空间，DQN 确实无从下手。** DQN 的突破不是说"不需要人"，而是说"人只需要搭建环境，不需要教策略"。以前做游戏 AI 等于既当裁判又当教练，DQN 让人只需要当裁判，教练的工作 AI 自己来。

这就好比：
- 以前教人打乒乓球：要教他什么叫上旋、什么叫下旋、怎么判断旋转……（人当教练）
- DQN 的方式：告诉他规则和怎么握拍，然后让他直接上桌打一万局（人只当裁判）

## 成绩单

- 在 49 款 Atari 游戏上用**同一套程序**（没有任何针对单个游戏的调整）
- 其中 6 款超过所有之前的方法
- 其中 3 款（打砖块、赛车、乒乓球）**打得比人类好**

## 这和前面的学习有什么关系？

前面 README 引用它，是为了说明整篇文档最重要的论点：

> **一个模型的智能（Agency）是训练出来的，不是用代码编出来的。**

DQN 没有"if 球在左边 then 向左"这样的规则。智能是从几千万次对战中自己涌现出来的。

但 DQN 也同时证明了另一个同样重要的道理——智能需要一个栖居的环境才能运作。人需要搭建这个环境（Harness）：
- 给它记分牌（奖励函数）
- 给它手柄（动作空间）
- 给它眼睛（输入预处理）
- 给它训练场地（游戏环境）

**我们（Harness 工程师）的工作不是用代码写出智能。我们的工作是给智能搭建一个能运转的环境——让模型可以自己学会剩下的一切。**
